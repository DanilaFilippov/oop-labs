Лабораторная №9 ( ПОЛИМОРФИЗМ НА ОСНОВЕ ИНТЕРФЕЙСОВ )
1. При наследовании интерфейса класс берёт на себя обязательство реализовать методы, объявленные в интерфейсе. При наследовании реализации класс получает уже готовую реализацию методов от базового класса.
Ключевое отличие: в первом случае класс сам реализует поведение, во втором — использует готовую логику родителя.
2. Полиморфизм проявляется, когда мы работаем с переменной типа интерфейса, которая может ссылаться на объекты разных классов, реализующих этот интерфейс. В момент вызова метода будет выполняться та версия, которая определена в конкретном классе объекта.
3. Класс может реализовывать любое количество интерфейсов.
4. Интерфейсы объявляются с помощью ключевого слова interface. Они используются для определения контракта, который классы должны реализовать, и для обеспечения полиморфизма без необходимости в иерархии наследования.
5. Ошибок нет
6. Ошибка в строке ob2.Draw();. Класс Parent не реализует интерфейс IDraw и не имеет метода Draw(). Поэтому вызов ob2.Draw() недопустим.

Лабораторная №20 ( МЕХАНИЗМ ПРИВЯЗКИ WPF )
1. Механизм, который устанавливает соединение между свойствами двух объектов: целевого (обычно элемент UI) и источника (данные, другой элемент). При изменении одного свойства автоматически обновляется другое, согласно настроенному направлению .
2. Привязка объявляется через расширение разметки {Binding ...}. Основные элементы синтаксиса :
Path	- Путь к свойству источника (например, Path=Name)
Source - Прямое указание объекта-источника
ElementName - Привязка к другому элементу по его имени
RelativeSource - Привязка к элементу относительно текущего (Self, FindAncestor, TemplatedParent)
Mode	- Направление потока данных
Converter - Конвертер значений
3. OneWay – данные идут от источника к цели (только чтение).
TwoWay – синхронизация в обе стороны.
OneTime – однократное чтение из источника при создании привязки.
OneWayToSource – данные идут от цели к источнику (обратный OneWay).
Default – режим, определяемый целевым свойством (обычно TwoWay для редактируемых свойств, OneWay для остальных).
4. Схема: цель (целевое свойство зависимости) ← привязка ← источник (любое свойство). Основные элементы: целевой элемент, целевое свойство, объект привязки, источник привязки, свойство-источник.
5. Целевой элемент – элемент управления, у которого устанавливается привязка.
Элемент-источник – объект, откуда берутся данные.
Целевое свойство – свойство зависимости целевого элемента, которое обновляется.
Свойство-источник – свойство объекта-источника, с которым связываются.
6. Источником может быть любой объект, но целевое свойство обязательно должно быть свойством зависимости. При этом источник должен уведомлять об изменениях (реализовать INotifyPropertyChanged) для автоматического обновления интерфейса в режимах TwoWay или OneWay
Лабораторная №22 ( АСИНХРОННЫЕ ДЕЛЕГАТЫ )
1. IAsyncResult представляет состояние асинхронной операции. Он позволяет отслеживать завершение (свойство IsCompleted), ожидать через AsyncWaitHandle, а также получать дополнительные параметры через AsyncState
2. Thread.Sleep() приостанавливает выполнение текущего потока на указанное количество миллисекунд. Используется для имитации длительной работы, тестирования многопоточности или создания задержек
3. Асинхронный вызов делегата через BeginInvoke возвращает IAsyncResult. Для получения результата вызывается EndInvoke, который передаёт возвращаемое значение из метода, выполнявшегося асинхронно. Если метод не успел завершиться, EndInvoke блокирует поток до получения результата
4. В C# метод может вернуть только одно значение. Для возврата нескольких значений можно:
1.	использовать кортеж (Tuple или ValueTuple),
2.	определить собственный класс/структуру с полями для каждого значения,
3.	использовать выходные параметры (out), но в асинхронных делегатах out не поддерживается, поэтому предпочтительнее первые два способа
5. Пользовательский делегат – объявляется с помощью delegate и может быть асинхронно вызван через BeginInvoke.
Библиотечные делегаты (Action, Func) – готовые обобщённые делегаты, также поддерживают асинхронный вызов.
Лямбда-выражения – сокращённая форма записи анонимных методов, которые могут быть присвоены любому совместимому типу делегата (пользовательскому или библиотечному).
Они взаимозаменяемы: любой асинхронный метод можно вызвать как через пользовательский делегат, так и через Func/Action, а лямбда может быть использована для их инициализации. Однако для BeginInvoke нужен именно тип делегата (явный или через Func/Action), но сама логика вызова не отличается
